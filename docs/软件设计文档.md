## 2.3 软件系统介绍

### 2.3.1 软件整体介绍

#### 一、系统概述

本软件是一个基于 **华为云 IoTDA（IoT Device Access）物联网平台**的**车内儿童安全智能监控系统**，运行于 **HarmonyOS（OpenHarmony）** 智能终端设备（手机/平板）。系统采用"**端-云-端**"三层物联网架构，通过 MQTT over TLS/SSL 协议实现终端 App 与车载嵌入式传感器节点之间的双向实时通信，形成一套完整的「**感知 → 传输 → 分析 → 告警 → 控制**」闭环智能安全体系。

**应用场景**：当驾驶员离开车辆后，系统持续监测车内环境参数（温度、湿度、CO₂浓度、甲醛浓度）及人体红外感应状态。一旦检测到异常（如温度过高、有害气体超标、有儿童遗留），系统将通过手机通知栏推送告警，并支持远程控制车载风扇、车门舵机、蜂鸣器等执行器，最大程度保障车内儿童安全。

#### 四、关键技术指标

| 指标项 | 参数 | 说明 |
|--------|------|------|
| **通信协议** | MQTT v3.1.1 over TLS/SSL | 轻量级物联网协议，支持QoS 0/1 |
| **认证方式** | HMAC-SHA256 签名 | 设备密钥 + 时间戳 → 一次性密码 |
| **传输加密** | TLS 1.2 / GlobalSign CA | 8883端口，服务端证书验证 |
| **连接保活** | Keep-Alive 60s | 心跳间隔 |
| **断线重连** | 指数退避 + 随机抖动 | 默认重连开启，maxBackoff 可配置 |
| **数据刷新** | 设备影子拉取 (Pull) + 推送 (Push) | 主动查询 + 被动接收 |
| **视频流协议** | HTTP MJPEG | 局域网WiFi直连，低延迟 |
| **UI 框架** | ArkUI (声明式) | HarmonyOS NEXT 原生组件 |
| **状态管理** | AppStorage + @State + @StorageLink | 全局共享 + 局部响应式 |
| **设计模式** | 单例 / 观察者 / 策略 / 适配器 | 服务层解耦 |

#### 模块 4：状态监控页 (StatusPage) — 核心业务页面

**文件位置**：`entry/src/main/ets/pages/StatusPage.ets`

**功能说明**：系统的核心业务页面，负责展示传感器实时数据（CO₂、甲醛、温度、湿度、人体红外）、MQTT 设备连接状态管理，以及提供手动数据上报和视图模式切换功能。支持列表视图与仪表盘视图两种数据展示模式。

**设计要点**：
- 通过观察者模式（回调函数）接收 IoTDAService 的数据推送
- `aboutToAppear` 中注册回调并建立 MQTT 连接
- `aboutToDisappear` 中注销回调防止内存泄漏
- 双视图模式：列表视图展示详细数值和阈值判断，仪表盘视图展示直观的圆形仪表

**关键变量**：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `co2` | `@State string` | CO₂ 浓度值 (ppm)，如 "450" |
| `hcho` | `@State string` | 甲醛浓度 (mg/m³)，如 "0.05" |
| `temperature` | `@State string` | 温度值 (°C)，如 "26.5" |
| `humidity` | `@State string` | 湿度值 (%)，如 "55" |
| `motion` | `@State string` | 人体红外状态，"0"=无人 / "1"=有人 |
| `fanState` | `@State string` | 风扇状态，"0"=开 / "1"=关 |
| `doorState` | `@State string` | 车门状态，"0"=开 / "90"=关 |
| `connectionStatus` | `@State string` | MQTT 连接状态文本描述 |
| `isConnected` | `@State boolean` | 是否已连接 (控制 UI 状态) |
| `isDashboardMode` | `@State boolean` | 是否为仪表盘视图模式 |
| `uploadEnvTemp/Hum/CO2/TVOC` | `@State string` | 数据上报表单输入值 |
| `iot` | `IoTDAService` | IoT 服务单例引用 |

**核心函数流程**：

##### 4.1 页面初始化 — `aboutToAppear()`

```

│   输入: 无
│
├── 1. 注册传感器数据回调
│   │   iot.registerDataCallback((temp, humidity, co2, hcho, motion, fanState, doorState) => {
│   │     │   输入: 7 个传感器/控制状态参数 (均为 string)
│   │     │
│   │     ├── 更新 @State 变量
│   │     │   this.temperature = temp
│   │     │   this.humidity = humidity
│   │     │   this.co2 = co2
│   │     │   this.hcho = hcho
│   │     │   this.motion = motion
│   │     │   this.fanState = fanState
│   │     │   this.doorState = doorState
│   │     │
│   │     └── 存储历史数据
│   │         addSensorDataPoint(CO2, Number(co2))
│   │         addSensorDataPoint(TEMPERATURE, Number(temp))
│   │         addSensorDataPoint(HUMIDITY, Number(humidity))
```

```
│   │   输入: 无 (使用 selectedSensorType 全局状态)
│   │   处理:
│   │     1. chartDataPoints = getSensorDataByType(selectedSensorType, 15)
│   │        → 从 AppStorage 中获取指定类型最近 15 条数据
│   │        → 按时间戳升序排列
│   │     2. calcParams()     计算柱状图 Y 轴参数
│   │     3. calcLineParams() 计算折线图 Y 轴参数
│   │   输出: 更新 chartDataPoints, minVal, range, lineMinVal, lineMaxVal, lineRange
│   │
│   └── [用户切换传感器类型]
│        └── selectedSensorType 变更 → 触发 loadData() 重新计算
│
├── DataPage.calcParams()
│   │   输入: chartDataPoints (SensorDataPoint[] 数组)
│   │   处理:
│   │     1. 遍历所有数据点，找到 maxValue 和 minValue
│   │     2. range = maxValue - minValue
│   │     3. if (range === 0) range = 1  (防止除零)
│   │   输出: 更新 minVal, range
│   │
├── DataPage.calcLineParams()
│   │   输入: chartDataPoints (SensorDataPoint[] 数组)
│   │   处理:
│   │     1. 遍历找到 maxValue 和 minValue
│   │     2. diff = maxValue - minValue
│   │     3. lineMinVal = minValue - diff × 0.2  (下方留 20% 余量)
│   │     4. lineMaxVal = maxValue + diff × 0.2  (上方留 20% 余量)
│   │     5. lineRange = lineMaxVal - lineMinVal
│   │   输出: 更新 lineMinVal, lineMaxVal, lineRange
│   │
├── DataPage.getBarWidth(value: number): string
│   │   输入: value — 当前数据点的数值
│   │   处理:
│   │     ratio = (value - minVal) / range   计算相对比例 [0, 1]
│   │     widthPct = 10 + ratio × 60         映射到 10%~70% 范围
│   │   输出: 返回百分比宽度字符串，如 "45%"
│   │
└── DataPage.getLineBottomRatio(value: number): number
    │   输入: value — 当前数据点的数值
    │   处理:
    │     ratio = (value - lineMinVal) / lineRange
    │     return Math.max(0, Math.min(1, ratio))  约束在 [0, 1]
    │   输出: 底部空白占比 (用于从下往上定位圆点)
```

#### 模块 9：IoTDA 服务层 (IoTDAService) — 核心业务逻辑

**文件位置**：`entry/src/main/ets/service/IoTDAService.ets`

**功能说明**：系统的核心业务逻辑服务，采用**单例模式** (`getInstance()`)。负责与华为云 IoTDA 平台建立 MQTT 连接、管理设备影子数据的拉取与解析、处理平台下发的设备控制命令、提供传感器属性上报和风扇/车门远程控制接口。是 UI 层与 Backend SDK 之间的**中间适配层**。

**设计模式**：
- **单例模式**：全局唯一实例，避免重复连接
- **观察者模式**：通过回调函数 (Callback) 实现数据推送
- **适配器模式**：将 Backend SDK 的低级 API 封装为业务级接口

**关键变量**：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `instance` | `static IoTDAService` | 单例实例 |
| `device` | `IoTDevice \| null` | Backend SDK 设备实例 |
| `connected` | `boolean` | 连接状态标志 |
| `dataCallback` | `SensorDataCallback \| null` | UI 数据回调 |
| `connectionCallback` | `ConnectionCallback \| null` | 连接状态回调 |
| `commandCallback` | `DeviceCommandCallback \| null` | 设备控制命令回调 |
| `fanState` | `string` | 风扇状态缓存 ("0"/"1") |
| `doorState` | `string` | 车门状态缓存 ("0"/"90") |
| `monitorService` | `DeviceMonitorService` | 设备监控服务引用 |

**核心函数流程**：

##### 9.1 连接建立 — `connect()`

```
IoTDAService.connect(): Promise<boolean>
│   输入: 无 (内部读取 IoTDAConfig 配置常量)
│
├── 1. 防重复连接检查
│   │   if (connected && device !== null) → return Promise.resolve(true)
│   │
├── 2. 创建 IoTDevice 实例
│   │   device = new IoTDevice(
│   │     IoTDAConfig.SERVER_URI,           // ssl://host:8883
│   │     IoTDAConfig.DEVICE_ID,            // 6a2918decbb0cf6bb9638b99_147852369
│   │     IoTDAConfig.DEVICE_SECRET,        // lsj1234567
│   │     IoTDAConfig.CA_FILE_PATH          // CA 证书路径
│   │   )
│   │
├── 3. 初始化设备 → 建立 MQTT 连接
│   │   const result = await device.init()
│   │   │   内部调用: DeviceClient.connect()
│   │   │   处理: TLS/SSL 握手 + HMAC-SHA256 认证
│   │   │   输出: Promise<boolean>
│   │
├── 4. 连接成功后的初始化
│   │   if (await device.client.isConnected()) {
│   │
│   │     // 4a. 设置命令监听器
│   │     device.client.commandListener = {
│   │       onCommand(requestId, serviceId, commandName, paras) {
│   │         → handleDeviceCommand(requestId, serviceId, commandName, paras)
│   │       }
│   │     }
│   │
│   │     // 4b. 设置影子数据监听器
│   │     device.client.shadowListener = {
│   │       onShadow(requestId, shadowDataList) {
│   │         → handleShadowData(shadowDataList)
│   │       }
│   │     }
│   │
│   │     // 4c. 拉取初始设备影子数据
│   │     await fetchShadow()
│   │     │   ├── getShadow({ deviceId, service_id: "environment" })   // zhxy 服务
│   │     │   └── getShadow({ deviceId, service_id: "environment" })   // environment 服务
│   │
│   │     // 4d. 通知连接成功
│   │     connected = true
│   │     connectionCallback?.(true, "连接成功")
│   │     return true
│   │   }
│   │
│   └── 5. 连接失败处理
│       connected = false
│       connectionCallback?.(false, "设备未连接")
│       return false
```

##### 9.2 影子数据解析 — `handleShadowData(list)`

```
IoTDAService.handleShadowData(list: ShadowData[])
│   输入: list — 平台推送的设备影子数据数组
│         ShadowData {
│           device_id: string,
│           service_id: string,
│           reported: { properties: Record<string, Object> },
│           desired: { properties: Record<string, Object> }
│         }
│
├── 1. 初始化局部变量
│   │   let temp='', humidity='', co2='', hcho='', motion=''
│   │   let fanState='', doorState=''
│   │
├── 2. 遍历 ShadowData[]
│   │   for each sd in list:
│   │     │
│   │     ├── 取数据源 (优先级: reported > desired)
│   │     │   props = sd.reported?.properties || sd.desired?.properties || {}
│   │     │
│   │     └── 逐属性解析:
│   │         ├── props['FAN']         → fanState
│   │         ├── props['SG90']        → doorState
│   │         ├── props['temperature'] → temp
│   │         ├── props['humidity']    → humidity
│   │         ├── props['CO2']         → co2
│   │         ├── props['TVOC']        → hcho (甲醛以 TVOC 表示)
│   │         └── props['motion']      → motion
│   │
├── 3. 判断是否有有效数据
│   │   hasAny = [temp,humidity,co2,hcho,motion,fanState,doorState]
│   │            .some(v => v !== '')
│   │
├── 4. [hasAny === true] 通知 UI 层
│   │   dataCallback?.(temp, humidity, co2, hcho, motion, fanState, doorState)
│   │   │   输出: StatusPage 更新 @State 变量 → UI 重新渲染
│   │   │
│   │   └── 同时通知监控服务

│   │       monitorService.processEnvironmentData(envData)
│   │
└── 5. [hasAny === false] 不触发回调 (无数据变化)
```

##### 9.3 设备命令处理 — `handleDeviceCommand(requestId, serviceId, commandName, paras)`

```
IoTDAService.handleDeviceCommand(requestId, serviceId, commandName, paras)
│   输入:
│     requestId: string    — 命令请求 ID，用于响应匹配
│     serviceId: string    — 服务 ID (如 "environment")
│     commandName: string  — 命令名称
│     paras: object        — 命令参数 { target: string, status: number }
│
├── 1. 解析命令参数
│   │   const target = paras['target']   // "fan" | "door" | "buzzer"
│   │   const status = Number(paras['status'])
│   │
├── 2. 更新本地状态缓存
│   │   if (target === 'fan' && status >= 0)   → fanState = String(status)
│   │   if (target === 'door' && status >= 0)  → doorState = String(status)
│   │
├── 3. 通知 UI 层 (可选)
│   │   if (commandCallback && target !== '')
│   │       commandCallback(target, status)
│   │
└── 4. 响应平台 (确认命令执行)
    │   const rsp: CommandRsp = {
    │     result_code: 0,                    // 0 = 成功
    │     response_name: commandName,
    │     paras: paras
    │   }
    │   device.client.respondCommand(requestId, rsp)
    │   输出: MQTT Publish 到 $oc/devices/{id}/sys/commands/response/...
```

#### 模块 10：Backend SDK (DeviceClient) — MQTT 通信核心

**文件位置**：`Backend/src/main/ets/client/DeviceClient.ets`

**功能说明**：华为云 IoT 设备侧 SDK 的核心 MQTT 客户端，封装了与 IoTDA 平台的所有底层 MQTT 通信逻辑。基于 `@ohos/mqtt` 原生模块实现，提供连接认证（HMAC-SHA256）、Topic 分发订阅、属性上报/设置/查询、命令响应、设备影子操作、断线重连等完整能力。

**设计模式**：
- **策略模式**：`functionMap` 中注册不同的 Handler 处理不同类型的 Topic 消息
- **观察者模式**：通过 Listener 接口组将平台事件通知上层
- **模板方法**：`AbstractDevice` 定义设备生命周期骨架，`IoTDevice` 提供具体实现

**关键常量**：

| 常量 | 值 | 说明 |
|------|-----|------|
| `SDK_VERSION` | `ARKTS_v0.0.1` | SDK 版本号，连接后上报 |
| `MESSAGE_DOWN_TOPIC` | `/messages/down` | 消息下行 Topic 后缀 |
| `COMMAND_DOWN_TOPIC` | `/sys/commands/request_id` | 命令下行 Topic 后缀 |
| `PROPERTY_SET_TOPIC` | `/sys/properties/set/request_id` | 属性设置 Topic 后缀 |
| `PROPERTY_GET_TOPIC` | `/sys/properties/get/request_id` | 属性查询 Topic 后缀 |
| `SHADOW_RESPONSE_TOPIC` | `/desired/properties/get/response` | 期望属性响应 Topic 后缀 |
| `SHADOW_GET_RESPONSE_TOPIC` | `/sys/shadow/get/response/request_id` | 影子查询响应 Topic 后缀 |
| `EVENT_DOWN_TOPIC` | `/sys/events/down` | 事件下行 Topic 后缀 |
| `CONNECT_BAD_USERNAME_OR_PASSWORD` | `4` | MQTT 错误码：认证失败 |

**关键变量**：

**核心函数流程**：

##### 10.1 MQTT 连接 — `connect()`

```
DeviceClient.connect(): Promise<boolean>
│   输入: 无 (使用 clientConf 中的 serverUri, deviceId, secret)
│
├── 1. 检查已有连接状态
│   │   if (mqttClient && isConnected()) → resolve(true)  直接返回
│   │   if (mqttClient && !isConnected()) → destroy()     销毁旧连接
│   │
├── 2. 生成连接参数
│   │   timestamp = IoTUtil.getTimestamp()         // 当前 UTC 时间戳
│   │   clientId = `${deviceId}_0_1_${timestamp}`  // MQTT Client ID
│   │   password = CryptoJS.HmacSHA256(secret, timestamp).toString()
│   │              ↑ HMAC-SHA256 签名，作为一次性密码
│   │
├── 3. 创建 MQTT 客户端实例
│   │   mqttClient = MqttAsync.createMqtt({
│   │     url: serverUri,           // ssl://b2305098be.st1.iotda-device...
│   │     clientId: clientId,
│   │     persistenceType: 1        // 持久化类型
│   │   })
│   │
├── 4. 配置 MQTT 连接选项
│   │   const options: MqttConnectOptions = {
│   │     keepAliveInterval: 60,           // 心跳间隔 60s
│   │     userName: deviceId,              // 用户名 = 设备 ID
│   │     password: HMAC-SHA256 签名,       // 一次性密码
│   │     connectTimeout: 30,              // 连接超时 30s
│   │     MQTTVersion: 0,                  // MQTT v3.1.1
│   │     sslOptions: {
│   │       enableServerCertAuth: true,    // 验证服务端证书
│   │       trustStore: CA 证书路径         // GlobalSign-rootca.pem
│   │     }
│   │   }
│   │
├── 5. 执行连接
│   │   try {
│   │     await mqttClient.connect(options)
│   │   }
│   │
│   ├── [成功] → isConnected() === true
│   │   │
│   │   ├── messageArrived()   注册消息接收监听
│   │   ├── connectLost()      注册断线监听
│   │   ├── connectComplete()  触发连接完成回调
│   │   ├── reportDeviceInfo() 上报 SDK 版本信息
│   │   └── resolve(true)
│   │
│   └── [失败] → catch(err)
│       │
│       ├── err.code === 4 (认证失败)
│       │   └── reject(err)  不进行重连
│       │
│       └── 其他错误
│           └── reconnect()  自动重连
│
└── 输出: Promise<boolean>
```

**HMAC-SHA256 认证流程**：

```
[设备 ID]  [设备密钥]              [当前时间戳]
    │           │                       │
    └───────────┼───────────────────────┘
                │
                ▼
    ┌─────────────────────────┐
    │ CryptoJS.HmacSHA256(     │
    │   secret,                │
    │   timestamp              │
    │ ).toString()             │
    └───────────┬─────────────┘
                │
                ▼
    ┌─────────────────────────┐
    │ MQTT CONNECT 报文:       │
    │   username: deviceId     │
    │   password: SHA256 结果   │
    └───────────┬─────────────┘
                │
                ▼
    ┌─────────────────────────┐
    │ IoTDA 平台验证:           │
    │ 1. 用相同算法计算签名     │
    │ 2. 比对 password          │
    │ 3. 验证时间戳在有效窗口内  │
    └─────────────────────────┘
```

##### 10.2 消息接收与分发 — `messageArrived()`

```

│   注册: mqttClient.messageArrived(callback)
│
├── 接收 MQTT 消息
```

##### 10.6 断线重连 — `reconnect(retryTimes)`

```
DeviceClient.reconnect(retryTimes: number = 0): Promise<boolean>
│   输入: retryTimes — 当前重试次数 (从 0 开始递增)
│
├── 1. 检查自定义重连策略
│   │   if (customOptions.customBackoffHandler) {
│   │     customBackoffHandler.backoffHandler()  // 使用自定义策略
│   │     return false
│   │   }
│   │   if (!customOptions.reConnect) return false  // 禁止重连
│   │
├── 2. 计算退避等待时间 (指数退避 + 随机抖动)
│   │   getWaitTImeUntilNextRetry(retryTimes)
│   │   │
│   │   │   lowBound = backoff × 0.8
│   │   │   highBound = backoff × 1.0
│   │   │   randomJitter = random(lowBound, highBound)
│   │   │   powParam = retryTimes & 0x0F   (取低4位)
│   │   │   backoffWithJitter = 2^powParam × (randomJitter + lowBound)
│   │   │   waitTime = min(minBackoff + backoffWithJitter, maxBackoff)
│   │   │
│   │   │   示例: backoff=1000ms, retryTimes=0 → ~1000ms
│   │   │         backoff=1000ms, retryTimes=3 → ~8000ms
│   │   │         backoff=1000ms, retryTimes=6 → maxBackoff (上限)
│   │   │
│   │   └── 输出: waitTime (毫秒)
│   │
├── 3. 等待后重试
│   │   await sleep(waitTime)
│   │   try {
│   │     if (await this.connect()) → return true  // 重连成功
│   │   } catch (e) { LogUtil.error(e) }
│   │
└── 4. 递归重试
    │   retryTimes++
    │   return await this.reconnect(retryTimes)
    └── 输出: Promise<boolean>
```

#### 模块 13：历史数据管理 (HistoryData)

**文件位置**：`entry/src/main/ets/common/HistoryData.ets`

**功能说明**：管理传感器历史数据点和操作历史记录的存储与查询。基于 HarmonyOS `AppStorage` 实现全局共享，所有页面通过 `@StorageLink` 绑定后自动同步更新。

**数据结构定义**：

```typescript
// 历史操作记录
interface HistoryRecord {
  id: number;          // 唯一标识 (时间戳)
  timestamp: string;   // 格式化时间 "2026-07-01 09:30:00"
  description: string; // 事件描述文字
}

// 传感器数据点
interface SensorDataPoint {
  id: number;          // 唯一标识 (时间戳)
  timestamp: number;   // 毫秒时间戳
  type: SensorType;    // 传感器类型枚举
  value: number;       // 测量数值
}

// 传感器类型枚举
enum SensorType {
  CO2 = 'CO2',
  HCHO = 'HCHO',
```

### 附录 C：错误处理与异常流程

```
═══════════════════════════════════════════════════════════════════════════
MQTT 连接异常处理流程
═══════════════════════════════════════════════════════════════════════════

[连接失败]
    │
    ├── err.code === 4 (认证失败)
    │   └── reject(err) → 不重连 → 通知用户检查设备凭证
    │
    ├── 网络超时
    │   └── reconnect(retryTimes=0)
    │       └── 指数退避重试 (最多 maxBackoff)
    │
    └── TLS 握手失败
        └── reject(err) → 通知用户检查 CA 证书

═══════════════════════════════════════════════════════════════════════════
MQTT 断线处理流程
═══════════════════════════════════════════════════════════════════════════

[connectLost 事件触发]
    │
    ├── connectionListener.connectionLost()  (通知上层)
    │
    ├── 检查自定义重连策略
    │   └── customBackoffHandler → 使用自定义策略
    │
    ├── 检查 reConnect 开关
    │   └── false → 不重连
    │
    └── reconnect(retryTimes=0)
        └── 指数退避 + 随机抖动重连

═══════════════════════════════════════════════════════════════════════════
数据上报异常处理
═══════════════════════════════════════════════════════════════════════════

[reportProperties / reportEnvironmentData]
    │
    ├── device 或 client 为 null
    │   └── return Promise.reject("mqtt client is null")
    │
    ├── isConnected() === false
    │   └── return Promise.reject("设备未连接")
    │
    └── publish() 失败
        └── catch(err) → 记录错误日志 → 不重试 (QoS 0)
```
